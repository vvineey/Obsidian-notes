

# 1. 접근제어 

정보보안 정책에 따라 허가된 사용자만 시스템 자원에 접근하도록 제한하는 것 

-> 데이터베이스는 외부 공격뿐 아니라 **권한을 가진 내부 사용자의 권한 남용**도 관리해야 함 


# 2. 접근제어 수행 요소 

### 주체 (Subject)
객체에 접근하려는 대상 

- 사용자 / 사용자 그룹
- 응용프로그램
- 실행 중인 프로세스 
- 데이터베이스 계정 

### 객체 (Object)
주체가 접근하려는 보호 대상 

- 데이터베이스 
- 테이블
- 컬럼 
- 파일
- 서버 

### 접근권한 (Privilege)
주체가 객체에 수행할 수 있는 행위

|기호|권한|설명|
|---|---|---|
|R|Read|객체의 내용을 읽을 수 있음|
|W|Write|객체의 내용을 수정할 수 있음|
|X|Execute|프로그램이나 프로시저를 실행할 수 있음|
|D|Delete|객체나 데이터를 삭제할 수 있음|
|C|Create|객체를 생성할 수 있음|



# 3. 접근제어 절차 

	식별 -> 인증 -> 인가 

### 식별 (Identification)
사용자가 시스템에 자신이 누군인지 알리는 과정

### 인증 (Authentication)
사용자의 신원이 실제로 맞는지 검증하는 관정 

| 방식    | 예                  |
| ----- | ------------------ |
| 지식 기반 | 비밀번호, PIN          |
| 소유 기반 | 스마트카드, 보안 토큰, 휴대전화 |
| 생체 기반 | 지문, 얼굴, 홍채         |
### 인가 (Authorization)
인가는 인증된 사용자가 특정 객체에 대해 어떤 행위를 수행할 수 있느
