# 정보처리기사 실기 MOC

> 기준: Q-Net 출제기준 자료실 기준, 현재 적용 출제기준은 `정보처리기사(2026.1.1 ~ 2026.12.31)`이며 최종수정일자는 `2025.06.25`이다. 이전 기준은 `2023.1.1 ~ 2025.12.31`로 별도 제공된다.

## 시험 개요

- 실기 과목명: `정보처리 실무`
- 출제 형태: 필답형 중심
- 공부 방향: 공식 출제기준의 하위 영역을 기준으로 잡고, 이론은 `SQL`, `보안`, `OS/네트워크`, `테스트`, `요구사항/UML`을 우선 정리한다. `프로그래밍 언어 활용`은 이론보다 코드 트레이싱과 기출 풀이 중심으로 본다.

## 기출 과목 정리

| 우선순위 | 영역 | 기출에서 확인할 포인트 |
|---|---|---|
| 최우선 | 프로그래밍 언어 활용 | 코드 결과 추론 문제 비중이 높다. 손으로 실행 흐름을 따라가는 연습이 필요하다. |
| 최우선 | SQL 응용 | 단답형과 결과 추론 모두 나온다. JOIN, GROUP BY, 트랜잭션, 정규화는 반복 점검한다. |
| 최우선 | 소프트웨어 개발 보안 구축 | 공격 기법과 대응책, 시큐어 코딩 용어가 자주 섞여 나온다. |
| 최우선 | 응용 SW 기초 기술 활용 | OS/네트워크/프로토콜 용어가 단답형으로 잘 나온다. |
| 상 | 애플리케이션 테스트 관리 | 테스트 기법, 커버리지, 결함 생명주기를 연결해서 외운다. |
| 상 | 요구사항 확인 | UML과 요구사항 분석 용어가 빈출이다. 다이어그램 종류별 목적을 구분한다. |
| 중 | 데이터 입출력 구현 | DB 설계, 키, 무결성, 인덱스, 트랜잭션 개념을 SQL과 연결한다. |
| 중 | 서버프로그램 구현 | 모듈화, 응집도/결합도, 공통 모듈, 배치 개념을 정리한다. |
| 중 | 인터페이스 구현 | JSON/XML, API, EAI/ESB, 연계 방식 용어를 구분한다. |
| 보강 | 통합 구현 | 연계 모듈과 송수신 구조를 용어 중심으로 챙긴다. |
| 보강 | 화면 설계 | UI/UX 설계 산출물과 원칙을 짧게 정리한다. |
| 보강 | 제품소프트웨어 패키징 | 릴리즈, 형상관리, DRM, 매뉴얼 관련 용어를 마무리 회독에 넣는다. |

## 회독 루틴

- 1회독: `SQL 응용` + `응용 SW 기초 기술 활용` + `프로그래밍 언어 활용` 코드 트레이싱
- 2회독: `소프트웨어 개발 보안 구축` + `애플리케이션 테스트 관리` + `요구사항 확인`
- 3회독: 나머지 설계/구현/패키징 영역을 단답형 키워드 중심으로 정리
- 기출 풀이 후: 틀린 문제를 위 12개 영역 중 하나로 분류하고, 해당 노트에 오답 포인트를 누적
