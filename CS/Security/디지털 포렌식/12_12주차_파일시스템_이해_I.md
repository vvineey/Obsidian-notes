
# 1. 파일 시스템 

저장매체에 파일과 디렉터리를 어떻게 저장하고 관리할지 정한 논리 체계이다. 
사용자는 파일 탐색기에서 폴더와 파일을 확인하지만, 
실제 디스크에는 섹터와 클러스터, 메타데이터, 할당 정보가 복잡하게 저장되어 있다. 


# 2. 물리적 관점과 논리적 관점 

### 물리적 관점 
저장장치를 섹터 같은 실제 저장 단위로 보는 관점
HDD 플래터, SSD 메모리 셸, 섹터, 물리 주소, 디스크 원시 바이트
### 논리적 관점 
운영체제와 사용자가 인식하는 구조를 중심으로 보는 관점 
C/D 드라이브, 폴더, 파일, 파일명, 파일 경로 


# 3. 저장장치의 논리적 구조 

1. 물리 디스크
2. 파티션
3. 파일시스템
4. 디렉터리
5. 파일
6. 파일 데이터와 메타데이터

# 4. 파티션 

하나의 물리 저장 장치를 여러 개의 독립된 논리 저장 영역으로 나누는 것 
예를 들어 하나의 SSD를 C 드라이브와 D 드라이브로 나누어 사용할 수 있다.

파일시스템은 파일과 디렉터리를 관리하는 규칙으로, 파티션을 사용하는 관리 규칙이다. 


# 5. 파티션 테이블 

디스크에 어떤 파티션이 존재하는지 기록한 구조다. 
- MBR
- GPT

## MBR
MBR : Master Boot Record 

전통적인 파티션 관리 방법으로, 디스크의 첫 번째 섹터인 0번 섹터에 위치한다. 

|구성|역할|
|---|---|
|Boot Code|부팅 과정에서 실행되는 코드|
|Partition Table|파티션 위치, 크기, 종류 저장|
|Signature|MBR이 정상 구조인지 식별|
- 일반적으로 512바이트 
- 0x55AA 시그니처 저장 
- 최대 4개의 파티션 엔트리 
- 약 2TB 이상의 대용량 디스크 사용 제한 
- 파티션 정보가 디스크 앞부분제 집중됨 

## GPT
GPT : GUID Partition Table

 MBR을 대체하는 현대적인 파티션 방식이며 UEFI 기반 시스템에서 사용된다.
 
 - 대용량 디스크 지원
- 많은 파티션 지원
- 파티션별 GUID 사용 (GUID : 각 파티션을 고유하게 식별하는 값)
- 백업 파티션 테이블 제공
- CRC 기반 무결성 검사
- UEFI와 연결


# 6. 섹터 

저장장치의 물리적인 최소 저장 단위이다. 
512바이트, MBR도 512바이트인 하나의 섹터에 저장된다. 
사용자가 파일 탐색기에서 직접 보는 단위는 아님!! 물리 단위이다. 


# 7. 클러스터 

파일시스템이 파일을 저장하고 관리하기 위해 여러 섹터를 묶어 만든 논리적 저장 단위이다.

- 섹터 크기: 512바이트
- 클러스터당 섹터 수: 8개
- 클러스터 크기 : 512 × 8 = 4096바이트

포맷할 때 선택하는 할당 단위 크기가 클러스터 크기이다. (= 파일은 클러스터 단위로 저장된다.)
파일이 1바이트, 클러스터가 4096바이트이면 남은 공간은 4095바이트이다.

### 작은 클러스터

장점:
- 작은 파일의 공간 낭비 감소
- 슬랙 공간 감소

단점:
- 관리해야 하는 클러스터 수 증가
- 파일시스템 관리 정보 증가 가능

### 큰 클러스터

장점:
- 대용량 파일 처리에 유리할 수 있음
- 관리해야 할 클러스터 수 감소

단점:
- 작은 파일이 많으면 공간 낭비 증가
- 슬랙 공간 증가


# 8. 슬랙

파일이 할당된 마지막 클러스터를 모두 사용하지 못 하고 남은 공간 
파일시스템은 데이터를 바이트 단위가 아니라 클러스터 단위로 할당한다.
따라서 파일 크기가 클러스터 크기의 정확한 배수가 아니면 마지막 클러스터에 남는 공간이 생긴다.

- 클러스터 크기: 4096바이트
- 파일 크기: 6000바이트
- 필요한 클러스터: 2개
- 할당 공간: 8192바이트
- 슬랙: 8192 - 6000 = 2192바이트

슬랙에는 과거 저장되어있던 데이터의 일부가 남아 있을 수 있다.
