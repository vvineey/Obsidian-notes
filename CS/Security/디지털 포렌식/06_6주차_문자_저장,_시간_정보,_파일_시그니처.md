
# 1. Base64

바이너리 데이터를 문자 데이터로 표현하는 인코딩 방식이다.

이미지, 실행 파일, 압축 파일 같은 바이너리 데이터는  이메일, URL, HTTP 메시지처럼 텍스트 중심 환경에서 깨질 수 있다.
Base64는 이런 데이터를 안전하게 전송하기 위해 64개의 문자 집합으로 바꿔 표현한다.
Base64는 암호화가 아니다!! 누구나 다시 디코딩할 수 있다. 


# 2. 문자 코드 

문자를 컴퓨터에 저장할 수 있도록 숫자 코드로 매핑해서 저장할 때의 규칙이다. 

### ASCII

영문 알파벳, 숫자, 특수문자, 제어문자를 표현하기 위한 초기 표준 문자 코드 
기본 ASCII는 7비트로 128개 문자를 표현, 확장 ASCII는 8비트로 256개 문자를 표현 

| 문자    | 10진수 | 16진수 |
| ----- | ---- | ---- |
| A     | 65   | 41   |
| a     | 97   | 61   |
| 0     | 48   | 30   |
| Space | 32   | 20   |

### Unicode

전 세계 문자를 하나의 체계에서 표현하기 위한 문자 집합
영어, 한글, 한자, 일본어, 특수문자, 이모지 표현 가능 

### UTF-8

Unicode 문자를 저장하는 대표적인 인코딩 방식

- 가변 길이 인코딩 
- 영문 1바이트 
- 한글은 여러 바이트 
- 현대 웹과 운영체제에서 널리 사용

# 5. MS-DOS Date/Time

FAT 파일 시스템

저장매체에서 파일과 폴더의 위치를 관리하는 파일 시스템
USB 메모리, SD 카드, 오래된 Windows 시스템 등에서 많이 사용된다.

FAT 파일 시스템은 MS-DOS Date/Time 방식 사용 

일반적으로 32비트 값에서
- 하위 16비트: 시간
- 상위 16비트: 날짜


# 6. DOS Time 구조 

### 시간영역 

- 초 (0~4비트) : 초는 2초 단위로 저장됨 !! 따라서 저장된 값에서 2를 곱해야 함
- 분 (5~10비트)
- 시 (11~15비트)


### 날짜 영역 
- 일 (16~20비트)
- 월 (21~24비트)
- 년도 (25~31비트) : 연도는 1980년도를 기준으로 저장됨 !! 따라서 저장된 연도 값에 1980을 더해야 함


ex) 0x6B7F7ECD

- Big Endian  
  - 6B    7F    7E   CD
  
  - 시간 영역 0x7ECD 
  - 날짜 영역 0x6B7F
  
  
- Little Endian  
  - CD    7E    7F   6B

  - 시간 영역 0x7F6B = 0111 1111 0110 1011
  - 날짜 영역 0xCD7E = 1100 1101 0111 1110


# 7. FILETIME

NTFS는 FILETIME 방식을 사용한다. 

- 기준 시각: 1601년 1월 1일 UTC
- 단위: 100나노초
- 크기: 64비트

FAT의 DOS Time보다 훨씬 넓은 범위와 높은 정밀도를 제공한다.

|구분|DOS Time|FILETIME|
|---|---|---|
|주 사용 파일시스템|FAT|NTFS|
|기준 연도|1980년|1601년|
|시간 단위|초는 2초 단위|100나노초 단위|
|크기|보통 32비트|64비트|
|정밀도|낮음|높음|


# 8. 파일 시그니처 

파일 내부의 시작 부분, 헤더에 기록된 고유 패턴이다. 

|파일 형식|대표 시그니처|
|---|---|
|PDF|%PDF|
|PNG|89 50 4E 47|
|JPG|FF D8 FF|
|ZIP|50 4B 03 04|
|GIF|47 49 46 38|

파일 시그니처는 파일의 실제 내부 구조를 보여준다.
확장자는 사용자가 쉽게 바꿀 수 있지만 바뀌어도 파일 내부 구조는 그대로다.
확장자가 .jpg여도 내부가 %PDF로 시작하면 실제로는 PDF 파일이다.
