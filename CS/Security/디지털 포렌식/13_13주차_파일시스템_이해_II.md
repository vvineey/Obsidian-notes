
# 1. FAT 파일시스템 

FAT : File Allocation Table
파일이 저장된 클러스터의 연결 관계를 FAT 테이블에 기록하여 파일의 위치를 관리한다. 

# 2. 파일시스템의 구조 
| 영역              | 역할                                |
| --------------- | --------------------------------- |
| Boot Sector     | 파일시스템의 기본 구조 정보 저장                |
| Reserved Sector | Boot Sector 이후 FAT 영역 전까지의 예약된 영역 |
| FAT1            | 주 파일 할당 테이블                       |
| FAT2            | FAT1의 백업 테이블                      |
| Root Directory  | 최상위 디렉터리 정보 저장                    |
| Data Area       | 실제 파일 내용 저장                       |

## Boot Sector

FAT 파일시스템의 시작 영역이다.
BPB가  들어있다.

- BPB (BIOS Parameter Block)
  섹터 크기, 클러스터 크기, FAT 위치, Root Diretory 위치 같은 레이아웃 정보 

## FAT 테이블 

각 클러스터의 사용 상태와 연결 관계를 기록한다. 
파일이 여러 클러스터에 나뉘어 저장되어 있다면 FAT 테이블을 따라가며 다음 클러스터를 찾는다. 

- FAT{190} = 191
- FAT{191} = EOF

파일은 190번 클러스터에서 시작 -> 다음 데이터는 191번 클러스터에 존재 -> 191번 클러스터에서 끝 

### FAT Entry
|값|의미|
|---|---|
|0x00000000|사용하지 않는 빈 클러스터|
|다음 클러스터 번호|파일 데이터가 이어지는 위치|
|EOF 값|파일의 마지막 클러스터|
|Bad Cluster 값|손상되어 사용할 수 없는 클러스터|
- EOF :  0x0FFFFFFF 


# 3. 8.3 파일명 
- 파일명: 최대 8자
- 확장자: 최대 3자

# 4. LFN 

LFN : Long File Name
8.3 형식보다 긴 파일명을 저장하기 위한 방식이다.

- 여러 Directory Entry를 사용
- Unicode 기반으로 긴 파일명 저장
- 실제 8.3 파일 엔트리 앞에 배치
- LFN Entry의 속성값은 일반적으로 0x0F
- LFN Entry는 이름만 저장하고, 실제 파일 관리 정보는 마지막의 일반 Directory Entry가 담당


# 5. FAT 파일 삭제 원리 

FAT에서 파일을 삭제해도 Data Area의 실제 내용을 즉시 모두 지우지는 않는다.

### 1. Directory Entry 변경

파일의 Directory Entry 첫 번째 바이트를 0xE5로 변경한다.
이 값은 해당 파일이 삭제되었음을 나타낸다.


### 2. FAT 테이블 변경

파일이 사용하던 클러스터의 연결 정보를 해제한다.
해당 FAT Entry를 0x00으로 바꾸어 사용할 수 있는 빈 클러스터로 표시할 수 있다.
- FAT[5] = 0x00
- FAT[8] = 0x00
- FAT[9] = 0x00
