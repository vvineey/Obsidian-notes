
# 1. FAT 파일시스템 

FAT : File Allocation Table
파일이 저장된 클러스터의 연결 관계를 FAT 테이블에 기록하여 파일의 위치를 관리한다. 

# 2. 파일시스템의 구조 
| 영역              | 역할                                |
| --------------- | --------------------------------- |
| Boot Sector     | 파일시스템의 기본 구조 정보 저장                |
| Reserved Sector | Boot Sector 이후 FAT 영역 전까지의 예약된 영역 |
| FAT1            | 주 파일 할당 테이블                       |
| FAT2            | FAT1의 백업 테이블                      |
| Root Directory  | 최상위 디렉터리 정보 저장                    |
| Data Area       | 실제 파일 내용 저장                       |

## Boot Sector

FAT 파일시스템의 시작 영역이다.
BPB가  들어있다.

- BPB (BIOS Parameter Block)
  섹터 크기, 클러스터 크기, FAT 위치, Root Diretory 위치 같은 레이아웃 정보 

## FAT 테이블 

각 클러스터의 사용 상태와 연결 관계를 기록한다. 
파일이 여러 클러스터에 나뉘어 저장되어 있다면 FAT 테이블을 따라가며 다음 클러스터를 찾는다. 

- FAT{190} = 191
- FAT{191} = EOF

파일은 190번 클러스터에서 시작 -> 다음 데이터는 191번 클러스터에 존재 -> 191번 클러스터에서 끝 

### FAT Entry
|값|의미|
|---|---|
|0x00000000|사용하지 않는 빈 클러스터|
|다음 클러스터 번호|파일 데이터가 이어지는 위치|
|EOF 값|파일의 마지막 클러스터|
|Bad Cluster 값|손상되어 사용할 수 없는 클러스터|
- EOF :  0x0FFFFFFF 


# 3. 8.3 파일명 

- 파일명: 최대 8자
- 확장자: 최대 3자

# 4. LFN 

LFN : Long File Name
8.3 형식보다 긴 파일명을 저장하기 위한 방식이다.

- 여러 Directory Entry를 사용
- Unicode 기반으로 긴 파일명 저장
- 실제 8.3 파일 엔트리 앞에 배치
- LFN Entry의 속성값은 일반적으로 0x0F
- LFN Entry는 이름만 저장하고, 실제 파일 관리 정보는 마지막의 일반 Directory Entry가 담당


# 5. FAT 파일 삭제 원리 

FAT에서 파일을 삭제해도 Data Area의 실제 내용을 즉시 모두 지우지는 않는다.
### 1. Directory Entry 변경
파일의 Directory Entry 첫 번째 바이트를 0xE5로 변경한다.
이 값은 해당 파일이 삭제되었음을 나타낸다.
### 2. FAT 테이블 변경
파일이 사용하던 클러스터의 연결 정보를 해제한다.
해당 FAT Entry를 0x00으로 바꾸어 사용할 수 있는 빈 클러스터로 표시할 수 있다.
- FAT[5] = 0x00
- FAT[8] = 0x00
- FAT[9] = 0x00


# 6. NTFS

NTFS : New Technology File System 
Windows NT 계열에서 도입되었으며 현대 Windows의 주요 파일시스템이다.

|구분|FAT|NTFS|
|---|---|---|
|핵심 구조|FAT 테이블|MFT|
|파일 메타데이터|Directory Entry|MFT Entry|
|기본 파일명|8.3, LFN 지원|긴 Unicode 파일명|
|보안 권한|제한적|ACL 지원|
|저널링|제한적|지원|
|암호화|기본 기능 제한|EFS 지원|
|삭제 표시|0xE5 사용|MFT 상태 변경|
|주요 환경|이동식 저장장치|Windows 시스템 디스크|


# 7. MFT 

MFT : Master File Table
NTSF에서 각 파일과 디렉터리는 일반적으로 MFT entry를 가진다.

구성 
- Header
- Fixup 정보
- Attribute 목록
- 사용 상태
- 파일 참조 번호

# 8. NTFS Attribute

NTFS에서는 파일 정보가 Attribute 형태로 저장된다.

|Attribute|주요 내용|
|---|---|
|$STANDARD_INFORMATION|시간, 권한, 파일 속성|
|$FILE_NAME|파일명, 부모 디렉터리, 시간 정보|
|$DATA|파일의 실제 데이터|
|$INDEX_ROOT|디렉터리 인덱스 정보|
|$SECURITY_DESCRIPTOR|보안과 권한 정보|

같은 파일의 시간 정보가 여러 Attribute에 존재할 수 있다.
포렌식에서는 서로 다른 시간 정보가 일치하는지 비교해 타임스탬프 조작 가능성을 분석할 수 있다.



# 9. NTFS의 주요 기능 

### USN Journal
파일과 디렉터리의 변경 이력을 기록

### LogFile
파일시스템 작업을 기록하는 저널이다.
사용자 행동 전체를 장기간 기록하는 일반 활동 로그가 아닌
주로 NTFS 메타데이터 작업의 안정성과 복구를 위한 구조다.

### ADS (Altenate Data Stream)
하나의 파일에 기본 데이터 외의 추가 데이터 스트림을 저장할 수 있게 한다.
정상적인 용도로도 사용되지만 데이터 은닉에 악용 위험이 있다.

### EFS (Encrypting File System)
파일 단위 암호화를 제공한다.

### VSS (Volume Shadow Copy Service)
특정 시점의 파일이나 볼륨 상태를 섀도 복사본으로 저장할 수 있다.


|구분|FAT|NTFS|
|---|---|---|
|메타데이터 핵심|Directory Entry|MFT Entry|
|할당 관리|FAT 테이블|$Bitmap과 MFT|
|삭제 표시|첫 바이트 0xE5|MFT Entry 상태 변경|
|연결 정보|FAT Entry 해제|클러스터 사용 상태 변경|
|실제 데이터|덮어쓰기 전까지 남을 수 있음|덮어쓰기 전까지 남을 수 있음|
|추가 흔적|FAT와 Directory Entry|USN Journal, $LogFile 등|

# 10. 파일 카빙 

파일 카빙은 FAT, MFT, Directory Entry 같은 파일시스템 메타데이터를 사용하지 않고, 
파일의 시그니처와 내부 구조를 이용해 파일을 복구하는 기법이다.

- Directory Entry가 손상됨
- FAT 테이블이 손상됨
- MFT가 손상됨
- 파티션 정보가 손상됨
- 저장매체가 포맷됨
- 삭제 파일의 메타데이터가 사라짐
- 비할당 영역에 파일 데이터만 남음

파일시스템이 파일의 위치를 알려주지 못하면 원시 데이터에서 파일 시그니처를 찾아야 한다.

| 파일 형식 | 헤더 예시       |
| ----- | ----------- |
| JPG   | FF D8 FF    |
| PNG   | 89 50 4E 47 |
| PDF   | %PDF        |
| ZIP   | 50 4B 03 04 |
| GIF   | 47 49 46 38 |
+ JPG 파일은 일반적으로 FF D9로 끝난다
