
# 1. 활성화 시스템 분석

전통적인 디스크 포렌식 : 
저장매체를 분리하거나 이미징해서 비휘발성 데이터를 분석한다.

라이브 포렌식 :
전원이 켜져 있고 운영체제가 동작 중인 시스템을 대상으로 현재 상태를 수집하고 분석하는 것 
 
- 실행 중인 프로세스 
- 현재 네트워크 연결 
- 로그인 사용자 
- 열린 파일 
- 실행 중인 서비스 
- 메모리에 존재하는 악성코드 
- 복호화된 데이터와 암호키
- 클립보드 내용 


# 2. 라이브 포렌식 

실행 중인 시스템에서 데이터를 수집하는 포렌식 방식 

- 휘발성 데이터 확보
- 현재 침입 상태 확인
- 실행 중인 악성코드 탐지
- 공격자의 네트워크 연결 확인
- 로그인 사용자와 세션 확인
- 메모리 안의 증거 확보

현재 상태를 확인할 수 있다는 장점이 있지만, 분석 도구를 실행하는 행위 자체가 시스템 상태를 변경할 수 있음 
따라서 사용한 도구, 명령어, 실행 시각과 결과를 반드시 기록해야 함 


# 3. 휘발성 데이터/ 비휘발성 데이터 

### 휘발성 데이터 
전원이 꺼지거나 시스템이 재부팅되면 사라질 수 있는 데이터 

- RAM 내용 
- 실행 중인 프로세스 
- 현재 네트워크 연결 
- 로그인 세션 
- 열린 파일 
- 실행 중인 서비스 
- 클립보드 
- 명령어 기록 
- 시스템 시간 
- 메모리 안에 암호키 
- 메모리 기반 악성코드 

### 비휘발성 데이터 
전원이 꺼져도 저장매체에 남는 데이터 
상대적으로 나중에 수집 가능 -> 원본 보호를 위해 디스크 이미징과 해시 검증 필요 

- 문서, 사용자 파일 
- 파일시스템 메타데이터 
- Windows 이벤트 로그 
- 레지스토리 
- 브라우저 기록 
- 설치 프로그램 정보 
- 자동 실행 설정 
- 디스크 이미지 

|구분|휘발성 데이터|비휘발성 데이터|
|---|---|---|
|지속성|전원이 꺼지면 사라질 수 있음|전원이 꺼져도 남음|
|수집 우선순위|높음|상대적으로 낮음|
|대표 위치|RAM과 실행 중인 시스템|HDD, SSD, USB|
|예시|프로세스, 연결, 세션, 암호키|파일, 로그, 레지스트리|
|주요 분석|라이브 포렌식|디스크 포렌식|


# 4. 시스템 시간 기록 

라이브 포렌식을 시작할 때 시스템 시간을 기록해야 한다.

- 시스템에 표시되는 현재 시간
- 실제 표준 시간
- 시간대
- 시스템 시간과 실제 시간의 차이
- 명령 실행 시각

시스템 시간과 실제 시간이 다르면 로그와 파일 타임스템프를 보정해야 한다.
- date /t
- time /t

# 8. 프로세스 분석 

현재 어떤 프로그램이 실행 중인지 확인한다.
- 프로세스 이름
- PID
- 부모 프로세스

- 사용자 계정

# 9. 네트워크 연결 분석 

공격자가 현재 시스템에 접속했는지 판단하는데 중요하다. 

Windows 명령어 
- netstat -ano

옵션 :
- -a : 모든 연결과 수신 포트 표시  
- -n : 주소와 포트를 숫자로 표시 
- -o : 연결을 사용하는 PID 표시 

# 10. 사용자 계정과 로그인 세션 

공격자는 지속적으로 시스템에 접근하기 위해 신규 계정을 만들 수 있다. 

Windows 명령어
- net user
- net users
- query user
- whoami
- net localgroup administrators

# 11. 열린 파일과 서비스

### 열린 파일

실행 중인 프로그램이나 원격 사용자가 현재 사용 중인 파일을 확인할 수 있다.

관련 명령어와 도구:
- openfiles
- net file
- psfile

### 서비스

Windows 서비스는 백그라운드에서 계속 실행되는 프로그램이다.
악성코드는 재부팅 후에도 실행되도록 서비스를 등록할 수 있다.

# 13. WMIC

WMIC : Windows Management Instrumentation Command-line

Windows 시스템의 다양한 정보를 명령어로 조회할 수 있다. 
BIOS, CPU, 논리 디스크, 메모리, USB 장치, 계정, 프로세스, 서비스, 네트워크 설정, 설치 프로그램 정보를 얻을 수 있다.

WMIC는 필요한 속성만 선택하거나 조건을 지정할 수 있어 반복 수집과 자동화에 유용하다.


# 14. Sysinternals Suite

Sysinternals Suite는 Microsoft가 제공하는 고급 시스템 분석 도구 모음이다.

|도구|주요 기능|
|---|---|
|Process Explorer|프로세스와 부모·자식 관계 분석|
|Process Monitor|파일, 레지스트리, 프로세스 활동 실시간 추적|
|TCPView|네트워크 연결과 사용 프로세스 확인|
|Autoruns|자동 실행 항목 분석|
|PsService|Windows 서비스 정보 확인|
|PsFile|원격에서 열린 파일 확인|
|RAMMap|물리 메모리 사용 상태 분석|
|VMMap|프로세스별 가상 메모리 분석|

GUI 도구는 직관적이고, 명령어 도구는 자동화와 결과 저장에 유리하다.
