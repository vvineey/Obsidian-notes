
# 1. 활성화 시스템 분석

전통적인 디스크 포렌식 : 
저장매체를 분리하거나 이미징해서 비휘발성 데이터를 분석한다.

라이브 포렌식 :
전원이 켜져 있고 운영체제가 동작 중인 시스템을 대상으로 현재 상태를 수집하고 분석하는 것 
 
- 실행 중인 프로세스 
- 현재 네트워크 연결 
- 로그인 사용자 
- 열린 파일 
- 실행 중인 서비스 
- 메모리에 존재하는 악성코드 
- 복호화된 데이터와 암호키
- 클립보드 내용 


# 2. 라이브 포렌식 

실행 중인 시스템에서 데이터를 수집하는 포렌식 방식 

- 휘발성 데이터 확보
- 현재 침입 상태 확인
- 실행 중인 악성코드 탐지
- 공격자의 네트워크 연결 확인
- 로그인 사용자와 세션 확인
- 메모리 안의 증거 확보

현재 상태를 확인할 수 있다는 장점이 있지만, 분석 도구를 실행하는 행위 자체가 시스템 상태를 변경할 수 있음 
따라서 사용한 도구, 명령어, 실행 시각과 결과를 반드시 기록해야 함 


# 3. 휘발성 데이터/ 비휘발성 데이터 

### 휘발성 데이터 
전원이 꺼지거나 시스템이 재부팅되면 사라질 수 있는 데이터 

- RAM 내용 
- 실행 중인 프로세스 
- 현재 네트워크 연결 
- 로그인 세션 
- 열린 파일 
- 실행 중인 서비스 
- 클립보드 
- 명령어 기록 
- 시스템 시간 
- 메모리 안에 암호키 
- 메모리 기반 악성코드 

### 비휘발성 데이터 
전원이 꺼져도 저장매체에 남는 데이터 
상대적으로 나중에 수집 가능 -> 원본 보호를 위해 디스크 이미징과 해시 검증 필요 

- 문서, 사용자 파일 
- 파일시스템 메타데이터 
- Windows 이벤트 로그 
- 레지스토리 
- 브라우저 기록 
- 설치 프로그램 정보 
- 자동 실행 설정 
- 디스크 이미지 

|구분|휘발성 데이터|비휘발성 데이터|
|---|---|---|
|지속성|전원이 꺼지면 사라질 수 있음|전원이 꺼져도 남음|
|수집 우선순위|높음|상대적으로 낮음|
|대표 위치|RAM과 실행 중인 시스템|HDD, SSD, USB|
|예시|프로세스, 연결, 세션, 암호키|파일, 로그, 레지스트리|
|주요 분석|라이브 포렌식|디스크 포렌식|


# 4. 시스템 시간 기록 

라이브 포렌식을 시작할 때 시스템 시간을 기록해야 한다.

- 시스템에 표시되는 현재 시간
- 실제 표준 시간
- 시간대
- 시스템 시간과 실제 시간의 차이
- 명령 실행 시각

시스템 시간과 실제 시간이 다르면 로그와 파일 타임스템프를 보정해야 한다.
- date /t
- time /t

# 9. 네트워크 연결 분석 

공격자가 현재 시스템에 접속했는지 판단하는데 중요하다. 

Windows 명령어 : 
	netstat -ano

옵션 :
- -a : 모든 연결과 수신 포트 표시  
- -n : 주소와 포트를 숫자로 표시 
- -o : 연결을 사용하는 PID 표시
