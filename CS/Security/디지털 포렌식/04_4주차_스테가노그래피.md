
# 1. 스테가노그래피 

이미지, 음악, 영상, 문서 같은 디지털 콘텐츠 안에 비밀 데이터를 숨기는 기술이다.
겉으로는 평번한 파일로 보여서 숨겨진 데이터가 있다는 사실 자체를 감춘다.

- 이미지 안에 비밀번호 숨기기
- 음악 파일 안에 텍스트 숨기기
- 문서 파일 안에 압축 파일 숨기기
- 사진 파일 안에 악성코드나 명령어 숨기기

|구분|암호화|스테가노그래피|
|---|---|---|
|목적|내용을 읽지 못하게 함|정보 존재 자체를 숨김|
|겉보기|암호문처럼 수상해 보일 수 있음|일반 파일처럼 보임|
|핵심|기밀성|은닉성|
|예시|문서를 암호문으로 변환|이미지 안에 문서 숨김|

# 3. 이미지 기반 스테가노그래피 원리 

디지털 이미지는 픽셀로 구성됨
각 픽셀은 RGB 값 (Red/Green/Blue)으로 표현됨 
각 색상 값은 보통 0부터 255까지의 숫자로 표현되어 8비트로 저장됨 

예를 들어 어떤 픽셀의 RGB 값이 다음과 같으면

- R = 120
- G = 80
- B = 200

값이 아주 조금 바꿔도 사람의 눈으로는 차이를 알아차리기 어려움
이 특성을 이용해 데이터를 숨길 수 있음


# 4. LSB 방식 

LSB : Least Significant Bit
가장 낮은 자릿값을 가진 비트를 의미 

예를 들어 8비트 값이 
01111000

LSB 0을 1로 변경 -> 120에서 121이 됨 
사람 눈에는 거의 차이가 보이지 않음 그래서 이미지 RGB 값에서 LSB를 조금씩 바꾸어 비밀 데이터를 숨김 


# 5. 문자를 이미지에 숨기는 과정 

먼저 문자를 숫자로 바꿈 
H는 ASCII에서 72이고 2진수로는 0100100

이 8비트를 이미지 픽셀의 RGB 값 LSB에 나누어 저장함 

1. 숨길 문자를 ASCII 값으로 변환한다.
2. ASCII 값을 2진수로 바꾼다.
3. 이미지 픽셀의 RGB 값 LSB에 비트를 하나씩 저장한다.
4. 추출할 때는 LSB를 다시 읽어 8비트씩 묶는다.
5. 묶은 비트를 문자로 변환한다.


# 6. 종료 조건 

어디까지 숨겨진 메세지인지 알아야 된다.
종료 조건이 없으면 추출 프로그램은 이미지 전체에서 계속 비트를 읽어야 햔다..

- 메시지 끝에 특정 종료 패턴 삽입
- 숨겨진 메시지 길이를 앞부분에 저장
- 8개의 0비트 같은 종료 신호 사용


# 7. LSB 방식의 장점과 한계 

### 장점 
- 이해하기 쉬움 
- 구현이 비교적 간단함 
- 이미지 품질 변화 작음 

### 한계
- 통계적으로 탐지될 수 있음 
- 많은 데이터를 숨기면 이미지 품질이 변할 수 있음 
- 순차적으로 숨기면 패턴이 드러날 수 있음 
- 압축이나 변환 과정에서 숨겨진 데이터가 손상될 수 있음

# 11. 요약 

- 스테가노그래피는 정보가 있다는 사실 자체를 숨기는 기술이다.
- 암호화는 내용을 읽지 못하게 하는 기술이다.
- Carrier는 숨겨진 데이터를 담는 겉보기 파일이다.
- Payload는 실제로 숨겨지는 데이터다.
- LSB는 최하위 비트다.
- 이미지 LSB 방식은 픽셀 값의 미세한 변화를 이용한다.
- OpenStego는 숨기기와 추출 실습에 사용된다.
- 스테가노그래피는 이미지, 음성, 영상, 문서 등 다양한 파일에 적용될 수 있다.
- StegAnalysis는 숨겨진 데이터 존재 가능성을 탐지하는 분석이다.
- 스테가노그래피와 암호화, Base64를 구분해야 한다.
