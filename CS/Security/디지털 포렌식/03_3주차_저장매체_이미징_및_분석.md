# 1.  저장매체 이미징 

디지털 포렌식에서 저장매체는 가장 기본적인 증거 출처다.

저장매체
- HDD
- SSD
- USB 메모리
- SD 카드
- 외장하드
- 서버 디스크
- 스마트폰 저장공간
  

사용자는 직접 저장한 파일뿐 아니라 

- 삭제 파일 흔적
- 파일시스템 메타데이터
- 슬랙 공간
- 비할당 영역
- 로그
- 프로그램 실행 흔적
- 브라우저 기록
- 문서 열람 기록
  
같은 흔적을 남길 수 있다.

문제는 원본 저장매체를 직접 분석 -> 증거가 바뀔 가능성 있음 -> 이미지 사본 생성/ 사본 분석 



# 2. 디스크 이미징 

저장매체에 들어 있는 데이터를 비트 단위로 복제하는 과정 

일반 파일 복사 : 사용자가 보는 파일만 복사
디스크 이미징 : 저장매체 전체 구조를 가능한 한 그대로 복제, **Bit Stream Image**이라고도 함 

- 정상 파일
- 삭제 파일 흔적
- 파일시스템 구조
- 디렉터리 정보
- 비할당 영역
- 슬랙 공간
- 숨겨진 데이터
- 메타데이터

|구분|일반 파일 복사|디스크 이미징|
|---|---|---|
|대상|사용자가 볼 수 있는 파일 중심|저장매체 전체|
|삭제 파일|보통 포함되지 않음|흔적이 남아 있으면 포함 가능|
|비할당 영역|포함되지 않음|포함 가능|
|슬랙 공간|포함되지 않음|포함 가능|
|파일시스템 구조|제한적으로만 반영|구조 자체를 분석 가능|
|포렌식 적합성|낮음|높음|


# 3. 디스크 이미징 장점 

### 원본 보존 
원본 저장매체를 직접 분석하지 않고 사본에도 반복 분석 가능 

### 재현 가능성 
같은 이미지 파일을 다른 분석자가 다시 분석 -> 같은 결과 검증 가능 

### 법적 신뢰성 확보 
원본과 이미지의 해시값을 비교해 동일성 입증 가능 

### 숨은 흔적 분석 
삭제 파일, 슬랙 공간, 비할당 영역, 메타데이터까지 확인 가능 


# 4. 디스크 이미징과 라이브 포렌식의 차이 
| 구분     | 디스크 이미징              | 라이브 포렌식                |
| ------ | -------------------- | ---------------------- |
| 대상     | 비휘발성 저장매체            | 전원이 켜진 활성 시스템          |
| 주요 데이터 | HDD, SSD, USB, 파일시스템 | RAM, 프로세스, 네트워크 연결     |
| 장점     | 안정적, 반복 분석 가능        | 현재 상태와 휘발성 증거 확보 가능    |
| 단점     | 휘발성 데이터는 놓칠 수 있음     | 수집 행위가 시스템 상태를 바꿀 수 있음 |

# 5. 해시 검증 

이미징을 하면 반드시 해시값을 계산해야 함 

대표적인 해시 알고리즘 
- MD5
- SHA-1
- SHA-256

원본 저장매체의 해시값 = 이미지 파일의 해시값 -> 이미지가 원본과 동일하다는 근거가 됨 

- 해시값은 원본과 사본의 동일성을 확인한다.
- 해시값은 수집 이후 데이터가 바뀌지 않았는지 확인한다.
- 해시값은 파일 내용이 “진실”이라는 것을 증명하지 않는다.

해시값은 무결성(수집 이후 불변)과 원본성(원본 = 제출본)을 뒷받침하는 도구!

# 6. Write Blocker 

분석용 컴퓨터가 원본 저장매체에 데이터를 쓰지 못하도록 막는 장치 

이게 왜 필요할까?
운영체제는 저장장치를 연결하는 순간! 접근 시간, 인덱스 정보, 임시 파일, 시스템 메타데이터 등을 바꿀 수 있음 

따라서 write blocker는 원본 오염 가능성을 줄임 (증거 보존 장치의 역할)


# 7. FTK Imager

디지털 포렌식에 자주 쓰이는 GUI 기반 도구 

주요 기능 : 

- 물리 드라이브 이미지 생성
- 논리 드라이브 이미지 생성
- 이미지 파일 탐색
- 삭제 파일 확인
- Hex 보기
- MAC Time 확인
- 이미지 생성 후 해시 검증
- 증거 정보 기록

FTK로 이미징할 때 기록할 수 있는 정보 : 

- Case Number
- Evidence Number
- Examiner
- Description
- Notes


# 8. dd 명령어 

*dd*는 리눅스/유닉스 계열에서 디스크를 비트 단위로 복제할 때 사용하는 기본 도구다.

예시:
	dd if=/dev/sdb of=image.img bs=16K

의미:
- if :  input file, 원본 입력 대상
- of : output file, 이미지 출력 대상
- bs : block size, 한 번에 읽고 쓸 블록 크기

입출력을 반대로 지정 -> 원본을 덮어쓸 수 있음, 위험 !!

# 9. Autopsy

Autopsy는 오픈소스 디지털 포렌식 분석 도구다.

FTK Imager가 이미지 생성과 기본 탐색에 강하다면,
Autopsy는 수집된 이미지를 체계적으로 분석하는 데 유용하다.

주요 기능:

- 디스크 이미지 분석
- 파일시스템 분석
- 삭제 파일 복구
- 타임라인 분석
- 웹 기록 분석
- 쿠키 분석
- 이메일 분석
- 채팅 기록 분석
- 이미지/비디오 썸네일 확인


# 10. EXIF, MAC Time, 파일 시그니처

### EXIF

사진 파일에 저장되는 메타데이터

예시:

- 촬영 장비
- 촬영 시간
- GPS 정보
- 편집 프로그램
- 이미지 생성 정보

### MAC Time

파일의 시간 정보

- M: Modified, 수정 시간
- A: Accessed, 접근 시간
- C: Created, 생성 시간

파일의 생성, 수정, 접근 시간을 보면 사용자의 행동 흐름을 추론할 수 있다.

### 파일 시그니처

파일 내부의 시작 부분에 있는 고유 패턴

예시:

- PDF: %PDF
- PNG: 89 50 4E 47
- JPG: FF D8 FF
- ZIP: 50 4B 03 04

확장자는 속일 수 있지만 파일 시그니처는 실제 내부 구조를 보여준다.


# 11. HPA와 DCO

### HPA

HPA : Host Protected Area
일반 운영체제에서 보이지 않도록 숨겨진 디스크 영역

### DCO

DCO : Device Configuration Overlay
디스크의 일부 용량이나 기능을 숨기거나 조정할 수 있는 영역이다.

수집 도구가 HPA/DCO를 포함하느냐에 따라 이미지 결과와 해시값이 달라질 수 있음!!
