

**BIOS/UEFI → MBR/GPT → VBR → 부트 로더 → Windows 실행 → 사용자 활동 → 아티팩트 생성**


# 1. 아티팩트 

사용자의 행동이나 운영체제의 동작으로 생성된 흔적 

Windows의 대표 아티팩트:

- 레지스트리
- 이벤트 로그
- Prefetch
- UserAssist
- 최근 문서
- 자동 실행 항목
- 폴더 접근 기록
- USB 연결 기록
- 브라우저 방문 기록
- 다운로드 기록
- 캐시와 쿠키


# 2. BIOS

BIOS : Basic Input/Output System

컴퓨터 전원이 켜지면 하드웨어를 초기화, 운영체제를 불러올 준비를 한다. 

- CPU와 메모리 점검 
- 저장장치 확인 
- 키보드 등 기본 장치 초기화 
- 부팅 가능한 장치 탐색 
- 부트 코드 실행 


# 3. UEFI

BIOS를 발전시킴 펌웨어 인터페이스이다. 

- GUI 기반 설정 화면 제공 가능
- 마우스 사용 가능
- 대용량 디스크 지원
- GPT 지원
- 빠른 부팅
- Secure Boot 지원
- 네트워크 기능 확장
- 다양한 드라이버 지원

| 구분        | BIOS     | UEFI           |
| --------- | -------- | -------------- |
| 사용자 인터페이스 | 텍스트 중심   | GUI 지원 가능      |
| 주요 파티션 방식 | MBR      | GPT            |
| 대용량 디스크   | 제한적      | 지원             |
| 보안 기능     | 제한적      | Secure Boot 지원 |
| 부팅 속도     | 상대적으로 느림 | 상대적으로 빠름       |
| 입력 방식     | 키보드 중심   | 키보드와 마우스 지원 가능 |



# 4.  부팅 프로세스 

전통적인 BIOS와 MBR 환경의 흐름 

1. BIOS가 하드웨어를 초기화한다.
2. 부팅 가능한 저장장치를 찾는다.
3. 디스크의 MBR을 읽는다.
4. MBR이 활성 파티션을 찾는다.
5. 해당 파티션의 VBR을 실행한다.
6. VBR이 Boot Loader를 실행한다.
7. Boot Loader가 운영체제 커널을 메모리에 올린다.
8. 운영체제가 사용자 환경을 준비한다.

**BIOS/UEFI -> MBR → VBR → Boot Loader → Kernel Loading**

### Windows 부팅 내부 단계 

|구성요소|역할|
|---|---|
|Boot Manager|부팅할 운영체제와 모드를 결정|
|BCD|부팅 구성 정보를 저장|
|Winload|커널과 핵심 드라이버를 메모리에 로드|
|Kernel|운영체제의 핵심 기능 초기화|
|smss.exe|사용자 세션 준비|
|winlogon.exe|사용자 로그온 처리|
|explorer.exe|바탕화면과 파일 탐색기 환경 제공|

포렌식 관점에서는 BCD, Winload 흔적, 이벤트 로그, 레지스트리, 세션 정보가 의미 있는 분석 대상이 된다.


# 5. MBR 

MBR : Master Boot Record 

전통적인 MBR 디스크의 첫 번째 섹터인 0번 섹터에 위치한다. 

구성 요소 : 
- Boot Code
- Partition Table
- Signature


파티션 정보를 확인하고 활성 파티션의 VBR로 실행 제어를 넘긴다. 



# 6. VBR

VBR : Volume Boot Record 

각 파티션 또는 볼륨의 시작 영역에 존재, 해당 볼륨의 부팅에 필요한 정보를 담는다. 

주요 기능 : 
- 파일시스템 정보 확인
- 운영체제 부트 로더 위치 확인
- Boot Loader 실행

MBR이 부팅할 파티션을 찾는다면, VBR은 해당 파티션에서 운영체제 로더를 실행하는 역할을 한다. 



# 7. 레지스트리 

Windows의 중앙 설정 데이터베이스로 계층적 구조를 가지며 Key와 Value로 데이터를 저장한다.

- 운영체제 설정
- 하드웨어 정보
- 설치된 프로그램
- 사용자 환경 설정
- 서비스 설정
- 자동 실행 프로그램
- 파일 연결 정보
- USB 연결 흔적
- 최근 실행 및 접근 흔적
등이 저장된다. 

### 레지스트리 구조 

|구성|의미|
|---|---|
|Hive|레지스트리의 최상위 논리 단위|
|Key|폴더와 비슷한 구조|
|Subkey|Key 안에 있는 하위 Key|
|Value|실제 설정이나 데이터|
|Data|Value에 저장된 내용|
ex) HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run



# 8. 주요 Hive

|Hive|의미|주요 정보|
|---|---|---|
|HKLM|HKEY_LOCAL_MACHINE|시스템과 하드웨어 정보|
|HKCU|HKEY_CURRENT_USER|현재 로그인 사용자 설정|
|HKU|HKEY_USERS|모든 사용자 프로파일 정보|
|HKCR|HKEY_CLASSES_ROOT|확장자와 프로그램 연결|
|HKCC|HKEY_CURRENT_CONFIG|현재 하드웨어 프로파일|
### HKLM 
시스템 전체에 적용되는 설정을 저장한다.
드라이버, 서비스, SAM, SECURITY, SOFTWARE, SYSTEM 같은 중요한 하이브와 연결된다.
### HKCU
현재 로그인한 사용자의 환경 설정이다. 
바탕화면, 프로그램 설정, 사용자별 자동 실행, 최근 사용 흔적 등이 연결된다.
### HKU
시스템에 존재하는 모든 사용자 프로파일 정보를 포함한다.
### HKCR
파일 확장자와 프로그램의 연결 정보를 관리한다. 
.txt 파일은 어떤 프로그램으로 열지, 어떤 명령으로 실행되는지가 저장되어 있다. 
### HKCC 
현재 사용 중인 하드웨어 프로파일 정보가 저장된다. 




# 9. UserAssist

사용자가 실행한 프로그램의 흔적을 저장하는 레지스트리 아티팩트
ROT13방식으로 인코딩되어 저장되는 경우가 있다. 

- 실행 프로그램
- 실행 횟수 
- 마지막 실행 시간 
- 실행 경로 
  를 확인할 수 있다. 


# 10. Prefetch

프로그램 실행 속도를 높이기 위해 Windows가 생성하는 파일 

- 실행 파일 이름
- 실행 횟수
- 최근 실행 시간
- 실행 과정에서 참조한 파일과 DLL
  를 확인할 수 있다. 

Prefetch가 존재 -> 프로그램이 실행되었을 가능성이 높다.

# 11. Run 키 

Windows 시작이나 사용자 로그온 시 자동으로 프로그램을 실행하기 위해 사용

- HKCU의 Run: 현재 사용자 자동 실행
- HKLM의 Run: 시스템 전체 사용자 자동 실행 

# 12. RecentDocs

사용자가 최근에 열었던 문서나 파일의 흔적을 저장 

- 최근 접근한 파일명
- 파일 확장자별 접근 흔적
- 최근 문서 사용 순서

# 13. OpenSavePidlMRU

파일 열기 또는 저장 대화상자에서 접근한 파일과 경로 흔적을 저장

# 14. LastVisitedPidlMRU

애플리케이션이 마지막으로 방문한 폴더나 경로 정보를 확인하는 데 사용

OpenSavePidlMRU, LastVisitedPidlMRU는 
사용자가 어떤 프로그램에서 어떤 파일이나 폴더에 접근했는지 추론하는 데 도움이 된다.

# 15. BagMRU

사용자가 탐색기로 열어본 폴더와 보기 설정에 대한 정보를 저장 

- 폴더 열람 흔적
- 하위 폴더 접근 흔적
- 폴더 보기 방식
- 삭제된 폴더의 과거 접근 흔적
- USB 내부 폴더 접근 흔적


# 16. TypedPaths

사용자가 파일 탐색기 주소창에 직접 입력한 경로를 저장

- 로컬 폴더
- 네트워크 공유 경로
- USB 경로
- 숨겨진 폴더 경로

# 17. USBSTOR

시스템에 연결된 USB 저장장치 정보를 저장

- 장치 제조사
- 제품명
- 모델
- 시리얼 번호
- 연결된 USB 종류

# 18. MountedDevices

저장장치와 드라이브 문자의 연결 관계를 확인하는 데 사용

- 특정 USB가 E: 드라이브로 연결됨
- 외장하드가 F: 드라이브로 연결됨


# 19. 웹 브라우저 아티팩트 

- 방문 기록
- 검색어
- URL
- 캐시
- 쿠키
- 다운로드 기록
- 자동완성 정보
- 로그인 정보
- 세션 정보

Chrome과 Edge 같은 브라우저는 SQLite 데이터베이스 형태로 기록을 저장
### 방문 기록 
방문한 URL, 페이지 제목, 방문 시각, 방문 횟수 
### 캐시 
웹페이지의 이미지, 스크립트, 문서 일부를 저장 
사용자가 방문 기록을 삭제하더라도 캐시가 남아 있음 -> 특정 웹사이트 방문 사실을 보조적으로 확인 가능 
### 쿠키 
웹사이트가 사용자 식별이나 세션 유지를 위해 저장하는 데이터 
접속한 도메인, 로그인 세션, 사용자 식별 값, 접속 유지 정보 등
### 다운로드 기록 
다운로드한 파일명, URL, 저장 경로, 시각, 성공 여부 등 


# 20. 아티팩트 교차 분석 

사용자가 USB의 문서를 열었다는 사실을 확인하기 

- USBSTOR: 어떤 USB가 연결되었는가?
- MountedDevices: 어떤 드라이브 문자였는가?
- BagMRU: USB 내부 폴더를 열었는가?
- RecentDocs: 어떤 문서를 열었는가?
- OpenSavePidlMRU: 어떤 경로에서 파일을 열었는가?
- 파일 시간 정보: 파일이 언제 접근되었는가?

이처럼 여러 흔적을 연결하면 단일 아티팩트보다 신뢰도 높은 결론을 만들 수 있다.
