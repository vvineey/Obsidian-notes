
# 1. 물리적 획득 

저장장치 전체를 가능한 한 비트 단위로 수집하는 방식이다.
파일시스템이 인식하는 파일뿐 아니라 삭제 데이터, 비할당 여역, 슬랙 공간까지 포함한다. 
디스크 이미징, 스마트폰이나 임베디드 장치에서는 CHIP-OFF, JTAG 같은 방식이 사용될 수 있다. 

- 장점 
삭제 데이터 흔적, 비할당 영역, 슬랙 공간, 메타데이터, 원시 데이터 등을 얻을 수 있다. 

- 단점
데이터 양이 매우 많고 수집 시간이 오래 걸리며 장치 손상 위험과 암호화된 데이터는 해석 어려움 

### CHIP-OFF
메모리 칩을 장치에서 물리적으로 분리한 뒤 데이터를 읽는 방식이다. 
장치를 물리적으로 분해하므로 **침습적인 획득 방법**이다.
침수, 파손, 고의 파괴 등으로 장치가 정상 부팅되지 않을 때 사용할 수 있다.

하지만 칩을 떼어내는 과정에서 장치가 손상될 수 있고, 전문 장비와 기술이 필요하다. 

절차는 칩 분리, 클리닝, 리더기 접합, 데이터 추출, 데이터 구조와 파일시스템 분석 순서이다.

### JTAG
칩의 디버깅 포트를 이용해 메모리에 접근하는 방식이다. 
칩을 분리하지 않고 접근할 수 있으므로 CHIP-OFF보다 비침습적이다. 
다만 장치에 따라 JTAG 포트가 숨겨져 있거나 비활성화되어 있을 수 있다.

JTAG와 CHIP-OFF는 모바일/임베디드 포렌식에서 중요한 개념이다. 
둘 다 일반 사용자가 파일 탐색기로 복사하는 방식과는 전혀 다르다.


# 2. 논리적 획득 

운영체제나 파일시스템이 인식하는 파일과 데이터를 중심으로 수집하는 방식이다. 
사용자가 보는 파일과 폴더, 애플리케이션 데이터, DB 등을 추출한다.

- 장점 
수집 속도가 빠르고 필요한 데이터만 선별 가능, 장치 손상 가능성 낮음, 비교 간단

- 단점 
운영체제가 허용하는 데이터만 수집 가능, 권한/잠금 상태 영향 받음


# 3. 스마트폰, 특수 장치 증거 수집 

### 스마트폰 
내장 메모리, SD 카드, USIM, 클라우드 동기화, 앱 데이터가 모두 관련될 수 있다. 
네트워크 연결을 차단하지 않으면 원격 삭제나 동기화로 데이터가 바뀔 수 있으므로 전자파 차단 포켓 같은 장비를 사용할 수 있다.

- 전자파 차단 포켓 
스마트폰이 이동통신, 블루투스, 와이파이 등 외부 네트워크와 통신하지 못하도록 차단하는 장비 

### 특수장치 
서버, 복합기, 프린터, CCTV, 블랙박스 등등 

# 4. 획득 시 고려사항

도구 별 수집 영역이 다르다!   
어떤 도구는 HPA/DCO를 포함하지 못할 수 있고, 어떤 도구는 파일시스템이 인식하는 영역만 수집할 수 있다. 
그래서 같은 장치라도 도구와 옵션에 따라 해시값이 달라질 수 있다.

따라서 보고서에는 사용한 도구, 버전, 옵션, 수집 대상, 포함/제외 영역을 명확히 써야 한다.


# 5. 검색 

디지털 증거는 양이 매우 많아서 검색 기술을 사용해 사건 관련 데이터를 줄여야 한다. 

### 일반 검색 
분석자가 키워드나 조건을 직접 입력해 관련 데이터를 찾는 방식

### 정규 표현식 검색 
일정한 패턴을 가진 문자열을 검색하는 방법

### 해시 검색 
파일의 해시값을 알려진 해시 데이터베이스와 비교하는 방식
정상 시스템 파일이나 알려진 프로그램 파일을 제외할 때 사용할 수 있다. 
NSRL 같은 데이터베이스가 예시다.

### 슬랙 검색 
파일이 클러스터 전체를 사용하지 못하고 남긴 공간을 분석하는 방식
슬랙 공간에는 과거 데이터 조각이나 삭제 흔적이 남을 수 있다. 


# 6. 통계 분석 

많은 데이터를 숫자로 요약해 비정상적인 패턴을 찾는 방법
파일 생성 시간이 특정 요일이나 시간대에 몰려 있는지, 특정 확장자가 비정상적으로 많은지, 로그인 실패가 특정 시간에 집중되는지 분석할 수 있다.


# 7. 시각화 

분석 결과를 그래프, 표, 타임라인 등으로 표현하는 것 

- 타임라인 시각화 
파일 생성/수정/접근 시간, 로그온 기록, 다운로드 기록 등을 시간순으로 정렬

검색, 통계, 시각화는 순서대로 연결된다. 
먼저 검색으로 관련 데이터를 줄이고, 통계로 패턴을 보고, 시각화로 사건 흐름을 설명한다.
