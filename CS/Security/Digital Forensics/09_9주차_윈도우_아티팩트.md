
# 1. 아티팩트 

사용자의 행동이나 운영체제의 동작으로 생성된 흔적 

Windows의 대표 아티팩트:

- 레지스트리
- 이벤트 로그
- Prefetch
- UserAssist
- 최근 문서
- 자동 실행 항목
- 폴더 접근 기록
- USB 연결 기록
- 브라우저 방문 기록
- 다운로드 기록
- 캐시와 쿠키


# 2. BIOS

BIOS : Basic Input/Output System

컴퓨터 전원이 켜지면 하드웨어를 초기화, 운영체제를 불러올 준비를 한다. 

- CPU와 메모리 점검 
- 저장장치 확인 
- 키보드 등 기본 장치 초기화 
- 부팅 가능한 장치 탐색 
- 부트 코드 실행 


# 3. UEFI

BIOS를 발전시킴 펌웨어 인터페이스이다. 

- GUI 기반 설정 화면 제공 가능
- 마우스 사용 가능
- 대용량 디스크 지원
- GPT 지원
- 빠른 부팅
- Secure Boot 지원
- 네트워크 기능 확장
- 다양한 드라이버 지원

|구분|BIOS|UEFI|
|---|---|---|
|사용자 인터페이스|텍스트 중심|GUI 지원 가능|
|주요 파티션 방식|MBR|GPT|
|대용량 디스크|제한적|지원|
|보안 기능|제한적|Secure Boot 지원|
|부팅 속도|상대적으로 느림|상대적으로 빠름|
|입력 방식|키보드 중심|키보드와 마우스 지원 가능|


# 4.  부팅 프로세스 

전통적인 BIOS와 MBR 환경의 흐름 

1. BIOS가 하드웨어를 초기화한다.
2. 부팅 가능한 저장장치를 찾는다.
3. 디스크의 MBR을 읽는다.
4. MBR이 활성 파티션을 찾는다.
5. 해당 파티션의 VBR을 실행한다.
6. VBR이 Boot Loader를 실행한다.
7. Boot Loader가 운영체제 커널을 메모리에 올린다.
8. 운영체제가 사용자 환경을 준비한다.

**BIOS/UEFI -> MBR → VBR → Boot Loader → Kernel Loading**

### Windows 부팅 내부 단계 

|구성요소|역할|
|---|---|
|Boot Manager|부팅할 운영체제와 모드를 결정|
|BCD|부팅 구성 정보를 저장|
|Winload|커널과 핵심 드라이버를 메모리에 로드|
|Kernel|운영체제의 핵심 기능 초기화|
|smss.exe|사용자 세션 준비|
|winlogon.exe|사용자 로그온 처리|
|explorer.exe|바탕화면과 파일 탐색기 환경 제공|

포렌식 관점에서는 BCD, Winload 흔적, 이벤트 로그, 레지스트리, 세션 정보가 의미 있는 분석 대상이 된다.


# 5. MBR 

MBR : Master Boot Record 

전통적인 MBR 디스크의 첫 번째 섹터인 0번 섹터에 위치한다. 

구성 요소 : 
- Boot Code
- Partition Table
- Signature


파티션 정보를 확인하고 활성 파티션의 VBR로 실행 제어를 넘긴다. 



# 6. VBR

VBR : Volume Boot Record 

각 파티션 또는 볼륨의 시작 영역에 존재, 해당 볼륨의 부팅에 필요한 정보를 담는다. 

주요 기능 : 
- 파일시스템 정보 확인
- 운영체제 부트 로더 위치 확인
- Boot Loader 실행

MBR이 부팅할 파티션을 찾는다면, VBR은 해당 파티션에서 운영체제 로더를 실행하는 역할을 한다. 



# 7. 레지스트리 

Windows의 중앙 설정 데이터베이스로 계층적 구조를 가지며 Key와 Value로 데이터를 저장한다.

- 운영체제 설정
- 하드웨어 정보
- 설치된 프로그램
- 사용자 환경 설정
- 서비스 설정
- 자동 실행 프로그램
- 파일 연결 정보
- USB 연결 흔적
- 최근 실행 및 접근 흔적
등이 저장된다. 

### 레지스트리 구조 

|구성|의미|
|---|---|
|Hive|레지스트리의 최상위 논리 단위|
|Key|폴더와 비슷한 구조|
|Subkey|Key 안에 있는 하위 Key|
|Value|실제 설정이나 데이터|
|Data|Value에 저장된 내용|
ex) HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run



# 8. 주요 Hive

|Hive|의미|주요 정보|
|---|---|---|
|HKLM|HKEY_LOCAL_MACHINE|시스템과 하드웨어 정보|
|HKCU|HKEY_CURRENT_USER|현재 로그인 사용자 설정|
|HKU|HKEY_USERS|모든 사용자 프로파일 정보|
|HKCR|HKEY_CLASSES_ROOT|확장자와 프로그램 연결|
|HKCC|HKEY_CURRENT_CONFIG|현재 하드웨어 프로파일|
### HKLM 
시스템 전체에 적용되는 설정을 저장한다.
드라이버, 서비스, SAM, SECURITY, SOFTWARE, SYSTEM 같은 중요한 하이브와 연결된다.
### HKCU
현재 로그인한 사용자의 환경 설정이다. 
바탕화면, 프로그램 설정, 사용자별 자동 실행, 최근 사용 흔적 등이 연결된다.
### HKU
시스템에 존재하는 모든 사용자 프로파일 정보를 포함한다.
### HKCR
파일 확장자와 프로그램의 연결 정보를 관ㄹ
