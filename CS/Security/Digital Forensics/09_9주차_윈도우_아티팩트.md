# 2. BIOS

BIOS : Basic Input/Output System

컴퓨터 전원이 켜지면 하드웨어를 초기화, 운영체제를 불러올 준비를 한다. 

- CPU와 메모리 점검 
- 저장장치 확인 
- 키보드 등 기본 장치 초기화 
- 부팅 가능한 장치 탐색 
- 부트 코드 실행 


# 3. UEFI

BIOS를 발전시킴 펌웨어 인터페이스이다. 

- GUI 기반 설정 화면 제공 가능
- 마우스 사용 가능
- 대용량 디스크 지원
- GPT 지원
- 빠른 부팅
- Secure Boot 지원
- 네트워크 기능 확장
- 다양한 드라이버 지원

|구분|BIOS|UEFI|
|---|---|---|
|사용자 인터페이스|텍스트 중심|GUI 지원 가능|
|주요 파티션 방식|MBR|GPT|
|대용량 디스크|제한적|지원|
|보안 기능|제한적|Secure Boot 지원|
|부팅 속도|상대적으로 느림|상대적으로 빠름|
|입력 방식|키보드 중심|키보드와 마우스 지원 가능|


# 4.  부팅 프로세스 

전통적인 BIOS와 MBR 환경의 흐름 

1. BIOS가 하드웨어를 초기화한다.
2. 부팅 가능한 저장장치를 찾는다.
3. 디스크의 MBR을 읽는다.
4. MBR이 활성 파티션을 찾는다.
5. 해당 파티션의 VBR을 실행한다.
6. VBR이 Boot Loader를 실행한다.
7. Boot Loader가 운영체제 커널을 메모리에 올린다.
8. 운영체제가 사용자 환경을 준비한다.

**MBR → VBR → Boot Loader → Kernel Loading**


# 5. MBR 

MBR : Master Boot Record 

전통적인 MBR 디스크의 첫 번째 섹터인 0번 섹터에 위치 
- Boot Code
- Partition Table
- Signature

파티션 정보를 확인하고 활성 파티션의 VBR로 실행 제어를 넘긴다.
