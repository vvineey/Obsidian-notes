# 1.  저장매체 이미징 ?

디지털 포렌식에서 저장매체는 가장 기본적인 증거 출처다.

저장매체
- HDD
- SSD
- USB 메모리
- SD 카드
- 외장하드
- 서버 디스크
- 스마트폰 저장공간
  

사용자는 직접 저장한 파일뿐 아니라 

- 삭제 파일 흔적
- 파일시스템 메타데이터
- 슬랙 공간
- 비할당 영역
- 로그
- 프로그램 실행 흔적
- 브라우저 기록
- 문서 열람 기록
  
같은 흔적을 남길 수 있다.

문제는 원본 저장매체를 직접 분석 -> 증거가 바뀔 가능성 있음 -> 이미지 사본 생성/ 사본 분석 



# 2. 디스크 이미징 

저장매체에 들어 있는 데이터를 비트 단위로 복제하는 과정 

일반 파일 복사 : 사용자가 보는 파일만 복사
디스크 이미징 : 저장매체 전체 구조를 가능한 한 그대로 복제, **Bit Stream Image**이라고도 함 

- 정상 파일
- 삭제 파일 흔적
- 파일시스템 구조
- 디렉터리 정보
- 비할당 영역
- 슬랙 공간
- 숨겨진 데이터
- 메타데이터

|구분|일반 파일 복사|디스크 이미징|
|---|---|---|
|대상|사용자가 볼 수 있는 파일 중심|저장매체 전체|
|삭제 파일|보통 포함되지 않음|흔적이 남아 있으면 포함 가능|
|비할당 영역|포함되지 않음|포함 가능|
|슬랙 공간|포함되지 않음|포함 가능|
|파일시스템 구조|제한적으로만 반영|구조 자체를 분석 가능|
|포렌식 적합성|낮음|높음|


# 3. 디스크 이미징 장점 

### 원본 보존 
원본 저장매체를 직접 분석하지 않고 사본에도 반복 분석 가능 

### 재현 가능성 
같은 이미지 파일을 다른 분석자가 다시 분석 -> 같은 결과 검증 가능 

### 법적 신뢰성 확보 
원본과 이미지의 해시값을 비교해 동일성 입증 가능 

### 숨은 흔적 분석 
삭제 파일, 슬랙 공간, 비할당 영역, 메타데이터까지 확인 가능 


# 4. 디스크 이미징과 라이브 포렌식의 차이 
| 구분     | 디스크 이미징              | 라이브 포렌식                |
| ------ | -------------------- | ---------------------- |
| 대상     | 비휘발성 저장매체            | 전원이 켜진 활성 시스템          |
| 주요 데이터 | HDD, SSD, USB, 파일시스템 | RAM, 프로세스, 네트워크 연결     |
| 장점     | 안정적, 반복 분석 가능        | 현재 상태와 휘발성 증거 확보 가능    |
| 단점     | 휘발성 데이터는 놓칠 수 있음     | 수집 행위가 시스템 상태를 바꿀 수 있음 |
