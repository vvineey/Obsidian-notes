

# 1. 컴퓨터 시스템 계층 

|계층|설명|예시|
|---|---|---|
|사용자|시스템을 사용하는 사람|문서 작성, 웹 검색, 파일 복사|
|애플리케이션|사용자가 실행하는 프로그램|브라우저, 메신저, 문서 편집기|
|운영체제|하드웨어와 프로그램을 관리|Windows, Linux, macOS|
|하드웨어|물리적인 시스템 구성요소|CPU, RAM, SSD, HDD|
사용자가 문서를 열면 애플리케이션 기록이 남고, 운영체제에는 최근 파일/레지스트리/로그가 남고, 저장장치에는 파일 데이터와 메타데이터가 남는다.

# 4. 주기억장치와 보조기억장치 

|구분|대표 장치|특징|
|---|---|---|
|주기억장치|RAM|빠르지만 전원이 꺼지면 데이터가 사라짐|
|보조기억장치|SSD, HDD, USB|전원이 꺼져도 데이터가 남음|
RAM은 CPU가 현재 실행 중인 프로그램과 데이터를 빠르게 사용하도록 돕는다.
SSD와 HDD는 운영체제, 프로그램, 사용자 파일, 로그를 장기간 저장한다. 

+ HDD : Hard Disk Drive 
+ SSD  : Solid State Drive

### 휘발성 메모리

- RAM 
- CPU 캐시 

### 비휘발성 메모리 
전원이 꺼져도 데이터가 남는 메모리 또는 저장장치 
- SSD 
- HDD
- USB
- SD 카드 
- ROM



# 6. 저장 원리에 따른 분류 

### 반도체 저장매체

- 빠른 속도
- 충격에 강한 편
- 이동식 장치에 많이 사용
- 전자 회로에 데이터 저장

### 자기 저장매체

- 가격 대비 대용량
- 기계식 구조
- 충격에 취약
- 삭제 데이터가 물리적으로 남을 수 있음

### 광학 저장매체

- 레이저로 데이터를 읽고 씀
- 오래된 백업이나 증거물에서 발견될 수 있음
- 현재 사용 빈도는 감소함

| 분류       | 저장 원리  | 예시                 |
| -------- | ------ | ------------------ |
| 반도체 저장매체 | 전자적 방식 | RAM, ROM, SSD, USB |
| 자기 저장매체  | 자기장 이용 | HDD, 플로피디스크        |
| 광학 저장매체  | 레이저 이용 | CD, DVD, Blu-ray   |



# 7. 플래시 메모리 

전원이 꺼져도 데이터가 남는 **비휘발성 반도체** 메모리 

- NOR 플래시 
- NAND 플래시 

### NOR 플래시 

읽기 속도가 빠르고 CPU가 직접 접근하기 쉽다. 

주요 용도:

- 펌웨어 : 하드웨어의 기본 작동 방법을 담은 내장 프로그램
- BIOS
- 임베디드 시스템 코드
- 부팅 코드

특징:

- 임의 접근에 유리
- 읽기 성능이 좋음
- 대용량 저장에는 상대적으로 불리
- 단가가 높은 편


### NAND 플래시 

대용량 저장에 적합하다. 

주요 용도:

- SSD
- USB 메모리
- SD 카드
- 스마트폰 내장 저장공간

특징:

- 대용량 저장에 유리
- 블록 단위로 데이터를 처리
- NOR보다 가격 대비 용량이 좋음
- 저장장치에 널리 사용

### Wear Leveling

NAND 플래시의 각 셀은 데이터를 쓰고 지울 수 있는 횟수 제한이 있다. 
웨어 레벨링은 특정 셀만 반복해서 사용하지 않도록 데이터를 여러 셀에 분산해 저장하는 기술이다. 

- SSD 수명 연장
- 특정 셀의 과도한 마모 방지
- 논리 주소와 물리 주소 불일치 가능성 
- 컨트롤러 내부 동작을 일반 도구로 직접 확인 어려움 


# 8. SLC, MLC, TLC

NAND 플래시는 하나의 셀에 몇 비트를 저장하는지에 따라 나뉜다.

|구분|셀당 저장 비트|특징|
|---|---|---|
|SLC|1비트|빠르고 수명이 길지만 비쌈|
|MLC|2비트|용량과 가격의 균형|
|TLC|3비트|저렴하고 대용량이지만 수명이 상대적으로 짧음|

일반적으로 한 셀에 더 많은 비트를 저장할수록 
- 용량 효율 증가
- 가격 저렴
- 쓰기 속도 감소 
- 내구성과 안정성 감소 



# 9. SSD

NAND 플래시를 사용하는 비휘발성 저장장치 

장점:

- 빠른 속도
- 소음이 적음
- 충격에 강함
- 전력 소비가 낮음
- 기계식 부품이 없음

포렌식상의 어려움:

- TRIM
- Garbage Collection
- 웨어 레벨링
- 컨트롤러 내부 동작

이 기능들 때문에 삭제된 데이터가 HDD보다 빠르게 사라지거나 물리적 위치가 바뀔 수 있다.

### TRIM 
운영체제가 SSD에 해당 영역의 데이터는 더이상 필요하지 않다고 알려주는 기능 
SSD는 이 정보를 바탕으로 삭제된 영역을 정리함 
이 과정에서 삭제된 데이터가 빠르게 제거, 복구 가능성 감소됨


# 10. HDD 

자기 플래터에 데이터를 기록하는 비휘발성 저장장치 

- 플래터
- 스핀들 모터
- 읽기/쓰기 헤드
- 액추에이터 암
- 컨트롤러

장점:

- 가격 대비 대용량
- 삭제 데이터 복구 가능성이 상대적으로 높을 수 있음
- 오랫동안 널리 사용됨

단점:

- 기계식 구조
- 충격에 취약
- SSD보다 느림
- 소음과 발열이 발생할 수 있음


|구분|SSD|HDD|
|---|---|---|
|저장 방식|NAND 플래시|자기 플래터|
|속도|빠름|상대적으로 느림|
|기계식 부품|없음|있음|
|충격 저항|강한 편|약한 편|
|삭제 복구|TRIM 등으로 복잡|상대적으로 가능성이 높을 수 있음|
|주요 이슈|웨어 레벨링, TRIM|물리 손상, 배드 섹터|




# 11. 저장장치 인터페이스 

저장장치는 특정 인터페이스를 통해 메인보드나 분석 장비에 연결된다. 

|인터페이스|주요 용도|
|---|---|
|SATA|일반 PC의 HDD와 SSD|
|M.2|메인보드에 직접 연결하는 SSD|
|NVMe|PCIe 기반 고속 SSD 통신 규격|
|SCSI|전문 장비와 서버|
|SAS|서버용 저장장치|
|USB|외장하드, USB 메모리 등 외부 장치|
포렌식 수집에서는 인터페이스를 알아야 적절한 어댑터, write blocker, 이미징 장비를 준비할 수 있다.



# 12. RAID 

여러 개의 디스크를 묶어 하나의 저장장치처럼 사용하는 방식 

- 속도 향상 
- 데이터 안정성 향상 
- 장애 대응 
- 저장공강 통합 


### RAID 0 : 속도 중심, 복제 없음 

스트라이핑 방식
데이터를 여러 디스크에 나누어 저장한다. 

장점:

- 읽기/쓰기 속도 향상
- 전체 용량을 효율적으로 사용

단점:

- 장애 복구 기능이 없음
- 디스크 하나만 고장 나도 전체 데이터가 손상될 수 있음

### RAID 1 : 복제 

미러링 방식
같은 데이터를 두 개 이상의 디스크에 복제한다. 

장점:

- 디스크 하나가 고장 나도 다른 디스크에 데이터가 남음
- 안정성이 높음

단점:

- 같은 데이터를 중복 저장
- 실제 사용 가능한 용량이 줄어듦


### RAID 5

스트라이핑 + 패리티를 결합한 방식 
포렌식에서는 디스크 순서와 스트라이프 크기, 패리티 규칙을 알아야 RAID 데이터를 재구성할 수 있다.

특징:

- 최소 3개 디스크 필요
- 데이터와 패리티를 분산 저장
- 디스크 하나의 장애를 복구할 수 있음
- 속도와 안정성의 균형

### RAID 10  : RAID 1의 안정성 + RIAD 0의 속도 

RAID 1 + RAID 0 

특징:

- 미러링과 스트라이핑 사용
- 속도와 안정성을 함께 확보
- 최소 4개 디스크 필요
- 사용 가능한 용량이 줄어듦
