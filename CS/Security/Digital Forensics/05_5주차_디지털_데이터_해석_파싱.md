
# 1.  데이터 해석 

디지털 포렌식은 원시 데이터 -> 의미 있는 증거로 바꾸는 작업임 

예를 들어 디스크 이미지 안에 다음과 같은 값이 있다고 하자.

- 41
- 6A
- FF D8 FF
- 50 4B 03 04

이 값들은 해석 방식에 따라 의미가 달라진다.

- 41은 16진수 숫자일 수도 있고, ASCII 문자 A일 수도 있다.
- 6A는 10진수로 106이다.
- FF D8 FF는 JPG 파일 시그니처일 수 있다.
- 50 4B 03 04는 ZIP 파일 시그니처일 수 있다.

즉, 포렌식 분석자는 데이터 자체뿐 아니라 **그 데이터를 어떤 규칙으로 해석해야 하는지** 알아야 한다.


# 2. 비트와 바이트

### 비트

비트는 컴퓨터가 다루는 가장 작은 정보 단위다.

- bit
- 값은 0 또는 1
- 참/거짓, 켜짐/꺼짐처럼 두 상태를 표현

### 바이트

바이트는 8개의 비트가 모인 단위다.

- 1 Byte = 8 bit
- 컴퓨터 저장장치는 보통 바이트 단위로 데이터를 저장하고 읽는다.

8비트로 표현할 수 있는 값은 다음과 같다.

- 2진수: 00000000부터 11111111
- 10진수: 0부터 255

그래서 RGB 색상 값도 보통 각 채널이 0부터 255까지의 범위를 가진다.

# 4. Endian

여러 바이트로 된 데이터를 저장할 때 **바이트 순서를 어떻게 배치하는지**를 의미한다.
4바이트 0x12345678를 메모리나 저장매체에 저장할 때 순서가 두 가지 방식으로 나뉠 수 있다. 
같은 바이트열도 Endian을 잘못 계산하면 완전히 다른 값이 되므로 
- 파일시스템 구조 분석
- 메모리 덤프 분석
- 시간 정보 파싱
- 디스크 구조 해석
- 네트워크 데이터 분석
- 바이너리 파일 포맷 분석
이 중요하다.

# 5. Big Endian 

Big Endian은 큰 자릿값 바이트를 먼저 저장하는 방식이다.
'0x12345678'을 Big Endian으로 저장하면 

'12 34 56 78 '이 된다. 
사람이 숫자를 읽는 순서와 비슷하다.

특징:
- 큰 자릿값이 앞에 온다.
- 네트워크 바이트 순서에서 자주 언급된다.
- 직관적으로 읽기 쉽다.

# 6. Little Endian

Little Endian은 작은 자릿값 바이트를 먼저 저장하는 방식이다.
'0x12345678'을 Little Endian으로 저장하면 

'78 56 34 12'이 된다.

Intel/AMD 계열 시스템에서 많이 사용된다.

# 8. 부호 - 크기 방식 

최상위 비트 MSB를 부호로 사용함 
- 0 : 양수 
- 1 : 음수 

00000101 = +5
10000101 = -5

- 이해하기 쉬운데 +0, -0 발생 

# 9. 1의 보수

양수의 비트를 모두 반전해서 음수로 표현 

00000101 = +5
11111010 = -5

- +0, -0 발생 


# 10. 2의 보수 

1. 양수의 비트를 모두 반전한다.
2. 1을 더한다.

00000101 = +5

비트 반전 
11111010

1 더하기 
11111011

- +0과 -0 문제가 없다.
- 덧셈 회로로 뺄셈을 처리하기 쉽다.
- 실제 컴퓨터에서 널리 사용된다.
