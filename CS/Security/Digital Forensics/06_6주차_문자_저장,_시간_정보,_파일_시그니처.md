
# 1. Base64

바이너리 데이터를 문자 데이터로 표현하는 인코딩 방식이다.

이미지, 실행 파일, 압축 파일 같은 바이너리 데이터는  이메일, URL, HTTP 메시지처럼 텍스트 중심 환경에서 깨질 수 있다.
Base64는 이런 데이터를 안전하게 전송하기 위해 64개의 문자 집합으로 바꿔 표현한다.
Base64는 암호화가 아니다!! 누구나 다시 디코딩할 수 있다. 


# 2. 문자 코드 

문자를 컴퓨터에 저장할 수 있도록 숫자 코드로 매핑해서 저장할 때의 규칙이다. 

### ASCII

영문 알파벳, 숫자, 특수문자, 제어문자를 표현하기 위한 초기 표준 문자 코드 
기본 ASCII는 7비트로 128개 문자를 표현, 확장 ASCII는 8비트로 256개 문자를 표현 

| 문자    | 10진수 | 16진수 |
| ----- | ---- | ---- |
| A     | 65   | 41   |
| a     | 97   | 61   |
| 0     | 48   | 30   |
| Space | 32   | 20   |

### Unicode

전 세계 문자를 하나의 체계에서 표현하기 위한 문자 집합
영어, 한글, 한자, 일본어, 특수문자, 이모지 표현 가능 

### UTF-8

Unicode 문자를 저장하는 대표적인 인코딩 방식

- 가변 길이 인코딩 
- 영문 1바이트 
- 한글은 여러 바이트 
- 현대 웹과 운영체제에서 널리 사용


# 3. 한글 인코딩 

한글 : 초성, 중성, 종성의 조합으로 이루어짐 

|구분|의미|
|---|---|
|조합형|초성, 중성, 종성을 조합해 글자를 표현|
|완성형|자주 쓰는 완성된 글자에 코드를 배정|
|확장 완성형|완성형의 표현 범위를 확장|
|EUC-KR|오래된 한국어 인코딩 환경에서 자주 사용|
|UTF-8|현대 시스템에서 널리 사용하는 유니코드 인코딩|
