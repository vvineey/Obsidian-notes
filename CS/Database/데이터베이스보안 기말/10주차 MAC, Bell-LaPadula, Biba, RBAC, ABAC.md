
# 1. 강제적 접근 제어 (MAC)

MAC : Mandatory Access Control 
중앙관리자가 주체와 객체에 보안등급을 무여하고, 미리 정해진 보안규칙에 따라 접근을 강제 

### MAC 특징
- 주체와 객체에 보안등급을 부여 
- 중앙관리자가 보안정책을 관리 
- 객체 소유자도 임의로 권한을 변경 불가능 
- 기밀성이 중요한 군사/공공기관에서 사용 
- **규칙 기반 접근제어**라고도 함

### MAC 장점 
- DAC보다 보안성이 높음 
- 사용자가 타인에게 임의로 권한 전달 불가 
- 중앙에서 보안정책을 일관되게 적용 가능 
- 민감정보 유출을 강력하게 통제 가능 

### MAC 단점 
- 모든 주체와 객체에 등급을 지정해야 함 
- 정책 설계와 구현이 어려움 
- 관리 부담이 큼 
- 사용자의 자유로운 업무 수행 제한 
- 변화하 많은 상업 환경에 적용 어려움
