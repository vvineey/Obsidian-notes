
# 1. 데이터베이스 암호화 생명 주기 

**데이터베이스 암호화 단계**
```
암호화 설계 → 암호화 구축 → 암호화 운영
```

|단계|주요 작업|
|---|---|
|설계|사용자 식별, 암호화 대상·알고리즘 선정, 권한 설계|
|구축|실제 데이터 암호화, 제약사항 유지, 키 관리체계 구축|
|운영|키 변경·백업·폐기, 접근통제, 감사, 성능 관리|


# 2. 데이이터베이스 암호화 설계 

데이터베이스 암호화를 구축하기 전 필요 사항을 식별하고 계획하는 단계

**설계항목**
- 데이터베이스 사용자 식별 
- 비인가 사용자 통제
- 복호화 권한 통제
- 암호화 대상 선정 
- 암호화 알고리즘 선정

# 3. 데이터베이스 사용자 식별 

암호화를 수행하기 전에 누가 데이터베이스를 사용하며, 어떤 방법으로 접근하는지 파악하는 단계 

**사용자 식별 정보**
- 사용자 계정
- IP 주소 
- 접속 날짜 시간
- 접속 도구 
- 접근 데이터 
- 데이터 변경 가능 여부 등등

**정형적인 접근**
특정 업무를 위해 정기적이고 반복적으로 데이터베이스에 접근하는 방식 
일반적으로 미리 정해진 SQL을 반복적으로 수행한다.

ex) 웹 서버, WAS, 업무 응용프로그램 등


**비정형적인 접근**
DBMS 유지보수나 개발을 위해 사용자가 데이터베이스에 직접 접근하는 방식 
다양한 SQL을 직접 실행하고 데이터를 조회·수정·삭제할 수 있으므로 더 강력한 접근통제와 감사가 필요하다.

ex) DBA, 개발자 등
