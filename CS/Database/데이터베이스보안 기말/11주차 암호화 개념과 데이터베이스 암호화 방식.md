
# 1. 암호화와 복호화 

## 평문 
암호화가 되지 않은 원본 데이터

## 암호화
평문의 의미를 알 수 없는 암호문으로 변환하는 과정, 데이터 **기밀성**을 보장  

```
평문 + 암호화 알고리즘 + 암호키 -> 암호문 
```

## 복호화 
암호문을 원래의 평문으로 복원하는 과정, 복호키 유출 시 복호화가 가능하여 암호키 관리가 중요 

```
암호문 + 복호화 알고리즘 + 복호키 -> 평문 
```

### 암호키 
암호화와 복호화 과정에 사용하는 특정 비트열 
동일한 알고리즘을 사용하더라도 키가 다르면 서로 다른 암호문이 됨

# 5. 대칭키 암호 

암호화키와 복호화키가 동일한 암호화 방식 

### 장점 
- 암호화와 복호화 속도가 빠르다.
- 대용량 데이터 암호화에 적합하다.
- 비대칭키 암호보다 비교적 적은 연산 비용이 필요하다.
- 데이터베이스, 파일, 통신 데이터 암호화에 널리 사용된다.

### 단점 
- 송신자, 수신자가 사전에 비밀키를 공유해야 한다.
- 키 전달 과정에서 유출 가능성이 있다.
- 사용자 수가 증가하면 관리해야 하는 키가 급증한다.
- 키가 유출되면 개노답이다.


### 대칭키 개수 계산 

```
n(n-1)/2
```




# 6. DES

DES : Data Encryption Standard 
과거에 널리 사용되던 블록 암호 알고리즘, 64비트 블록 암호 알고리즘, 키 길이 56비트 

|구분|크기|
|---|---|
|블록 크기|64비트|
|입력 키 크기|64비트|
|실제 암호화 키 길이|56비트|
|패리티 비트|8비트|


# 7. 3DES

3DES/ Triple-DES : DES의 짧은 키 보안 문제를 해결하기 위해 DES 연산을 세 번 수행한다.

```
EDE 방식 
DES 암호화 -> DES 복호화 -> DES 암호화 
```

- DES보다 보안성 좋음 
- 112비트 또는 168비트 수준의 키를 사용 
- 처리 속도는 느림
- 대부분 AES로 대체 


# 8. AES

AES : Advanced Encryption Standard 

DES를 대체하기 위해 선정된 표준 대칭키 블록 암호 알고리즘이다.

| 구분      | 크기              |
| ------- | --------------- |
| 블록 크기   | 128비트           |
| 지원 키 길이 | 128, 192, 256비트 |

AES는 키 길에 따라 수행하는 라운드 수가 달라짐

| 키 길이  | 라운드 수 |
| ----- | ----- |
| 128비트 | 10라운드 |
| 192비트 | 12라운드 |
| 256비트 | 14라운드 |


- DES, 3DES보다 보안성 높음 
- 처리 속도 빠름 
- 소프트웨어와 하드웨어에서 효율적 구현 가능
- 데이터베이스 암호화에 널리 사용됨 


# 9. DES  vs  3DES  vs  AES
|구분|DES|3DES|AES|
|---|---|---|---|
|블록 크기|64비트|64비트|128비트|
|키 길이|실제 56비트|112·168비트|128·192·256비트|
|처리 속도|빠름|느림|빠름|
|현재 안전성|취약|제한적 사용|널리 권장|
|특징|짧은 키 길이|DES를 세 번 수행|현재 표준 알고리즘|


# 10. 비대칭키 암호 

암호화키와 복호화키가 서로 다른 방식

사용자는 공개키와 개인키를 가진다.
- 공개키 : 다른 사람에게 공개할 수 있는 키 
- 개인키 : 소유자만 안전하게 보관해야 하는 키

### 장점 
- 사전에 동일한 비밀키를 공유할 필요 없음 
- 키 배포 문제 해결 
- 전자서명과 인증에 사용

### 단점 
- 대칭키 암호보다 연산 속도 느림 
- 대용량 데이터 전체를 암호화하기에는 비효율 
- 개인키 유출 -> 개노답 


# 11. 공개키 암호로 기밀성 통신 

송신자는 수신자의 공개키로 암호화하고 수신자는 자신의 개인키로 복호화한다.

```
송신자
→ 수신자의 공개키로 암호화
→ 암호문 전송
→ 수신자가 자신의 개인키로 복호화
```

# 12. 전자서명 

메세지가 실제 송신자가 작성했다는 사실과 메시지가 변경되지 않았음을 확인하기 위해 사용한다.

```
송신자
→ 자신의 개인키로 서명
→ 수신자가 송신자의 공개키로 서명 검증
```


# 13. RSA 

대표적인 비대칭키 암호 알고리즘 
큰 수의 소인수분해가 어렵다는 수학적 문제를 기반으로 한다.

- 대칭키 전달
- 전자서명
- 인증서 
- 소량 데이터 암호화 

RSA는 대칭키 암호보다 느리기 때문에 일반적으로 대용량 데이터를 직접 암호화하는 대신 
AES 키를 안전하게 전달하는 데 사용한다.


# 14. 하이브리드 암호 방식 

실제 시스템에서는 대칭키와 비대칭키 암호를 함께 사용한다.

1. 송신자가 임시 AES 대칭키를 생성한다.
2. 실제 데이터는 빠른 AES로 암호화한다.
3. AES 키는 수신자의 공개키로 암호화한다.
4. 수신자는 자신의 개인키로 AES 키를 복호화한다.
5. 복원한 AES 키로 데이터를 복호화한다.
