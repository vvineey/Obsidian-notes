
# 1. 빅데이터 

기존의 관리 방법이나 분석 도구로는 수집·저장·처리·분석하기 어려운 방대한 양의 정형·반정형·비정형 데이터와, 그 데이터에서 유용한 가치를 추출하기 위한 기술을 의미한다.

### 데이터 특성 관점
기존 방식으로 처리하기 어려운 대규모의 다양한 데이터 집합을 의미한다.

### 기술 관점
대규모 데이터를 수집, 저장, 처리, 분석, 시각화하여 가치 있는 정보를 만드는 기술과 아키텍처를 의미한다.


# 2. 데이터 유형 

### 정형 데이터 
미리 정해진 구조와 형식에 따라 저장되는 데이터

### 비정형 데이터 
완전히 고정된 테이블 구조는 아니지만, 태그나 키를 사용하여 데이터의 구조를 표현하는 데이터

| 구분    | 정형 데이터  | 반정형 데이터         | 비정형 데이터           |
| ----- | ------- | --------------- | ----------------- |
| 구조    | 고정된 스키마 | 부분적인 구조         | 일정한 구조 없음         |
| 대표 예  | RDB 테이블 | JSON, XML, HTML | 이미지, 영상, 텍스트      |
| 검색·분석 | 비교적 쉬움  | 추가 처리 필요        | 전처리와 특징 추출 필요     |
| 저장 방식 | 관계형 DB  | 문서 DB, 파일 등     | 분산파일시스템, 객체 저장소 등 |
| 특징    | 형식이 명확함 | 태그·키로 구조 표현     | 데이터 형태가 매우 다양함    |


# 3. 빅데이터의 3V
```
Volume + Velocity + Variety
```

### Volume
Volume은 데이터의 규모 또는 양을 의미한다.

데이터양이 매우 크기 때문에 단일 서버나 기존 데이터베이스만으로 저장·처리하기 어려울 수 있다.

**예시
- 수억 명의 SNS 게시물
- 전국 CCTV 영상
- 대형 쇼핑몰의 거래 기록
- IoT 센서가 지속적으로 생성하는 데이터

**필요한 기술
- 분산 저장
- 클라우드 스토리지
- HDFS
- 확장 가능한 데이터베이스

### Velocity
Velocity는 데이터가 생성·수집·처리되는 속도를 의미한다.

빅데이터는 빠르게 생성되므로 정해진 시간 안에 데이터를 처리해야 가치가 생긴다.

 **예시
- 실시간 금융 거래 탐지
- 실시간 교통정보 분석
- 보안 로그와 침입 탐지
- IoT 센서 이상 감지
- 실시간 상품 추천

**필요한 기술
- 스트림 처리
- 실시간 분석
- 메시지 큐
- 이벤트 기반 처리

### Variety
Variety는 데이터 형태의 다양성을 의미한다.

빅데이터는 관계형 데이터베이스의 정형 데이터뿐 아니라 반정형·비정형 데이터까지 포함한다.

**예시
- 정형: 고객정보 테이블, 거래 내역
- 반정형: JSON, XML, 로그
- 비정형: 이미지, 영상, 음성, SNS 텍스트


# 4. 빅데이터 처리 생명주기

```
수집 → 저장 → 처리 → 분석 → 표현·활용 → 폐기
```

수.저.처.분.표/활.폐




# 5. Hadoop
Hadoop은 대규모 데이터를 여러 서버에 분산하여 저장하고 처리할 수 있는 자바 기반 오픈소스 프레임워크이다.

일반 컴퓨터 여러 대를 연결하여 대규모 데이터를 처리할 수 있도록 설계되었다.

**장점**
- 대규모 데이터 분산 저장 가능
- 여러 서버에서 병렬 처리 가능
- 서버를 추가하여 시스템 확장 가능
- 일부 서버에 장애가 발생해도 처리 가능
- 비교적 낮은 비용으로 대규모 시스템 구축 가능

**구성요소**
- HDFS: 데이터 분산 저장
- MapReduce: 데이터 분산 처리

|기술|역할|
|---|---|
|Hadoop|대규모 데이터 분산 저장·처리 프레임워크|
|HDFS|Hadoop의 분산파일 저장 시스템|
|MapReduce|Hadoop의 분산 데이터 처리 방식|
|NoSQL|대규모·다양한 데이터를 유연하게 저장하는 데이터베이스|

# 6. 빅데이터 처리과정별 개인정보 보안 이슈

## 6-1. 수집 단계

### 문제
- 동의 없는 개인정보 수집
- 필요 이상의 과도한 수집
- 자동화된 프로그램에 의한 수집
- 권한 없는 주체의 데이터 수집

### 대응
- 수집 목적 명확화
- 최소수집
- 정보주체 동의
- 수집 권한 통제
- 안전한 통신 사용

## 6-2. 저장 단계

### 문제
- 대규모 개인정보 유출
- 데이터 변조
- 분산 저장환경의 관리 어려움
- 안전하지 않은 백업

### 대응
- 데이터 암호화
- 접근제어
- 데이터 등급 분류
- 무결성 검증
- 안전한 백업과 감사


## 6-3. 처리·분석 단계

### 문제
- 개인정보가 포함된 데이터를 그대로 분석
- 이용 목적 외 분석
- 여러 데이터를 결합하여 민감정보 추론
- 암호화되지 않은 데이터 처리

### 대응
- 비식별화된 데이터 사용
- 암호화된 데이터 처리
- 분석 목적 제한
- 분석 권한 분리
- 분석 과정 기록


## 6-4.  표현·활용 단계

### 문제
- 분석 결과의 무단 이용
- 외부 공개 과정에서 개인정보 노출
- 다른 데이터와 결합한 재식별

### 대응
- 공개 범위 제한
- 비식별화 적정성 평가
- 재식별 위험 검토
- 접근통제
- 제3자 제공 관리


## 6-5. 폐기 단계
### 문제
- 개인정보가 완전히 삭제되지 않음
- 백업·복제 데이터 잔존
- 저장장치 복구를 통한 유출

### 대응
- 복구 불가능한 논리적 삭제
- 물리적 저장매체 파기
- 암호키 폐기
- 백업과 복제본 동시 삭제
- 파기 결과 감사
