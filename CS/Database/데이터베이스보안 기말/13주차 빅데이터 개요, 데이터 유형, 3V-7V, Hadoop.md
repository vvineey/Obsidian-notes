
# 1. 빅데이터 

기존의 관리 방법이나 분석 도구로는 수집·저장·처리·분석하기 어려운 방대한 양의 정형·반정형·비정형 데이터와, 그 데이터에서 유용한 가치를 추출하기 위한 기술을 의미한다.

### 데이터 특성 관점
기존 방식으로 처리하기 어려운 대규모의 다양한 데이터 집합을 의미한다.

### 기술 관점
대규모 데이터를 수집, 저장, 처리, 분석, 시각화하여 가치 있는 정보를 만드는 기술과 아키텍처를 의미한다.


# 2. 데이터 유형 

### 정형 데이터 
미리 정해진 구조와 형식에 따라 저장되는 데이터

### 비정형 데이터 
완전히 고정된 테이블 구조는 아니지만, 태그나 키를 사용하여 데이터의 구조를 표현하는 데이터

| 구분    | 정형 데이터  | 반정형 데이터         | 비정형 데이터           |
| ----- | ------- | --------------- | ----------------- |
| 구조    | 고정된 스키마 | 부분적인 구조         | 일정한 구조 없음         |
| 대표 예  | RDB 테이블 | JSON, XML, HTML | 이미지, 영상, 텍스트      |
| 검색·분석 | 비교적 쉬움  | 추가 처리 필요        | 전처리와 특징 추출 필요     |
| 저장 방식 | 관계형 DB  | 문서 DB, 파일 등     | 분산파일시스템, 객체 저장소 등 |
| 특징    | 형식이 명확함 | 태그·키로 구조 표현     | 데이터 형태가 매우 다양함    |


# 3. 빅데이터의 3V
```
Volume + Velocity + Variety
```

### Volume
Volume은 데이터의 규모 또는 양을 의미한다.

데이터양이 매우 크기 때문에 단일 서버나 기존 데이터베이스만으로 저장·처리하기 어려울 수 있다.

### 예시

- 수억 명의 SNS 게시물
- 전국 CCTV 영상
- 대형 쇼핑몰의 거래 기록
- IoT 센서가 지속적으로 생성하는 데이터

### 필요한 기술

- 분산 저장
- 클라우드 스토리지
- HDFS
- 확장 가능한 데이터베이스
